Overview

This project was a comprehensive accessibility audit of the University of Michigan Biological Station website, conducted as part of SI 338. Working across six pages, I evaluated the site against WCAG 2.1 AA standards using a combination of automated tools, screen reader testing, and keyboard navigation, then produced a full report with prioritized remediation recommendations for each page.

01
Audit Setup

Defining scope and selecting tools

The audit covered six key pages of the UMBS website: Home, About Us, People, Researchers, Students, and Data, Maps & Site History. Testing combined automated tools and manual evaluation simultaneously, ensuring cross-verification and a comprehensive picture of accessibility performance across assistive technologies.

WCAG 2.1 AA WAVE Axe DevTools
02
Automated Testing

Running WAVE and Axe DevTools across all six pages

WAVE and Axe scans identified a range of structural and visual issues. The most frequent problem was content placed outside semantic landmarks — over 60 instances on the Students page alone — which prevents screen reader users from identifying and navigating between major page regions. Additional issues included contrast failures, missing and redundant alt text, empty links, and invalid heading hierarchies.

Axe DevTools identified 94 total issues across all six pages. Six were critical and 18 were serious, with the majority related to missing landmark structure.

WAVE Axe DevTools WCAG 2.1 AA
03
Manual Testing

Screen reader and keyboard navigation with VoiceOver

Automated tools only tell part of the story. VoiceOver and keyboard-only testing surfaced issues that scans cannot detect, including inconsistent focus visibility on mobile, interactive elements unreachable via keyboard, and accordion states announced only as "plus" or "minus" without indicating whether the section was open or closed. On the People page, keyboard users had to tab through every letter of the alphabet to reach the field of study filter.

VoiceOver Keyboard Navigation
04
Heuristic Analysis

Evaluating usability against Jakob Nielsen's 10 heuristics

Beyond WCAG compliance, the site was evaluated against Nielsen's 10 usability heuristics. Key gaps included inconsistent button styling with no clear visual hierarchy, insufficient system status feedback during form submissions and interactive transitions, no support for user preferences like reduced motion or dark mode, and no confirmation prompts when navigating to external links or downloading files.

Nielsen's 10 Heuristics Usability Evaluation
05
Remediation Planning

Prioritized recommendations for each page

Each of the six pages received a tailored remediation plan. Across all pages, the highest priority fix was implementing proper semantic HTML landmarks, adding header, main, nav, and footer elements to enclose all content regions. This single structural change would address the largest category of issues across the entire site. Additional recommendations covered heading hierarchy, alt text, color contrast, focus visibility, and keyboard access for interactive components like the data dashboard and profile filters.
